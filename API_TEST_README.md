# 🧪 API测试工具使用说明

## 概述

这是一个专为前端开发设计的API测试工具，可以在后端尚未完成时进行前端开发和测试。支持模拟API和真实API之间的无缝切换。

## 功能特性

### ✅ 支持的API模块
- **认证API**: 登录、注册、获取用户信息、更新用户
- **学期API**: 获取学期列表
- **课程API**: 课程CRUD操作
- **聊天API**: 聊天会话管理、消息发送
- **文件API**: 文件上传、获取文件列表
- **管理员API**: 用户管理、邀请码管理

### 🎯 核心功能
- **模拟数据**: 完整的模拟数据，包含用户、课程、聊天等
- **API切换**: 支持模拟API和真实API之间的切换
- **实时测试**: 可视化的测试界面，实时显示测试结果
- **错误处理**: 完善的错误处理和日志记录
- **文件上传**: 支持文件上传测试

## 快速开始

### 1. 环境配置

创建 `.env` 文件并配置以下环境变量：

```bash
# 使用模拟API（推荐用于开发阶段）
REACT_APP_USE_MOCK_API=true

# 或者配置真实API地址
REACT_APP_API_BASE_URL=http://localhost:8000/api
REACT_APP_USE_MOCK_API=false
```

### 2. 启动应用

```bash
npm start
```

### 3. 访问测试工具

在开发模式下，访问 `http://localhost:3000/api-test` 即可使用API测试工具。

## 配置说明

### 环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `REACT_APP_API_BASE_URL` | 真实API基础URL | `http://localhost:8000/api` |
| `REACT_APP_USE_MOCK_API` | 是否使用模拟API | `true` (当未设置API_BASE_URL时) |

### API切换逻辑

1. 如果 `REACT_APP_USE_MOCK_API=true`，强制使用模拟API
2. 如果设置了 `REACT_APP_API_BASE_URL` 且 `REACT_APP_USE_MOCK_API=false`，使用真实API
3. 如果未设置 `REACT_APP_API_BASE_URL`，默认使用模拟API

## 故障排除

### 常见问题

1. **测试失败**: 检查网络连接和API地址配置
2. **文件上传失败**: 确保文件大小在合理范围内
3. **认证失败**: 检查用户名密码是否正确

### 调试技巧

1. 查看浏览器控制台的错误信息
2. 检查测试日志中的详细错误信息
3. 确认环境变量配置正确

## 最佳实践

### 开发阶段
1. 使用模拟API进行前端开发
2. 定期运行API测试确保功能正常
3. 在提交代码前进行完整测试

### 联调阶段
1. 切换到真实API进行联调
2. 对比模拟API和真实API的差异
3. 及时更新模拟数据以匹配真实API

### 生产环境
1. 移除API测试页面的路由
2. 确保使用真实API地址
3. 关闭开发模式下的测试链接
